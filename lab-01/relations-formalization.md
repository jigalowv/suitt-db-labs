Між сутностями існують 3 зв’язки:
- N:1 (1:N)
- M:N
- 1:1

Тип зв’язку N:1 (Багато до одного)
- Багато contact до одного member: для реалізації цього зв’язку в таблиці contact був створений атрибут member_id (первинний ключ таблиці member).
- Багато contact до одного contact_type: для реалізації цього зв’язку в таблиці contact був створений атрибут contact_type_id (первинний ключ таблиці contact_type).
- Багато performer до одного income_range: для реалізації цього зв’язку в таблиці performer був створений атрибут expected_income_id (первинний ключ таблиці income_range).
- Багато performer_skill до одного performer: для реалізації цього зв’язку в таблиці performer_skill був створений атрибут performer_id (первинний ключ таблиці performer).
- Багато performer_skill до одного skill: для реалізації цього зв’язку в таблиці performer_skill був створений атрибут skill_id (первинний ключ таблиці skill).
- Багато required_skill до одного order: для реалізації цього зв’язку в таблиці required_skill був створений атрибут order_id (первинний ключ таблиці order).
- Багато required_skill до одного skill: для реалізації цього зв’язку в таблиці required_skill був створений атрибут skill_id (первинний ключ таблиці skill).
- Багато order до одного customer: для реалізації цього зв’язку в таблиці order був створений атрибут customer_id (первинний ключ таблиці customer).
- Багато order до одного income_range: для реалізації цього зв’язку в таблиці order був створений атрибут income_range_id (первинний ключ таблиці income_range).
- Багато perform_request до одного order: для реалізації цього зв’язку в таблиці perform_request був створений атрибут order_id (первинний ключ таблиці order).
- Багато perform_request до одного performer: для реалізації цього зв’язку в таблиці perform_request був створений атрибут performer_id (первинний ключ таблиці performer).
- Багато comment до одного order: для реалізації цього зв’язку в таблиці comment був створений атрибут finished_order_id (первинний ключ таблиці order).
- Багато comment до одного member (як автор): для реалізації цього зв’язку в таблиці comment був створений атрибут c_from (первинний ключ таблиці member).
- Багато comment до одного member (як отримувач): для реалізації цього зв’язку в таблиці comment був створений атрибут c_to (первинний ключ таблиці member).

Тип зв’язку M:N (Багато до багатьох)
- Багато performer до багатьох skill:
  для реалізації цього зв’язку була створена таблиця performer_skill з атрибутами:
    - ps_id - сурогатний ключ таблиці.
    - performer_id - первинний ключ таблиці performer.
    - skill_id - первинний ключ таблиці skill.
- Багато order до багатьох skill:
  для реалізації цього зв’язку була створена таблиця required_skill з атрибутами:
    - rs_id - сурогатний ключ таблиці.
    - order_id - первинний ключ таблиці order.
    - skill_id - первинний ключ таблиці skill.

Тип зв’язку 1:1 (Один до одного)
- Один member до одного performer: для реалізації цього зв’язку в таблиці performer був створений атрибут member_id (первинний ключ цієї таблиці, а також зовнішній ключ, що вказує на первинний ключ таблиці member).
- Один member до одного customer: для реалізації цього зв’язку в таблиці customer був створений атрибут member_id (первинний ключ цієї таблиці, а також зовнішній ключ, що вказує на первинний ключ таблиці member).
